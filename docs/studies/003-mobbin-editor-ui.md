# Study 003: Mobbin Editor UI References

Date: 2026-05-31

Relievo remains a full-screen browser instrument, not a productivity suite. The useful references from Mobbin were therefore canvas-first creative editors rather than dashboards.

## References

- [Visual Electric web screen](https://mobbin.com/screens/c93c648e-a9ea-46ff-8bcd-9d11f0ff7706): dark creative canvas with compact persistent actions.
- [Runway web screen](https://mobbin.com/screens/e474ed18-f79c-42ee-82ee-527e58183a40): media creation workspace with a restrained control surface around the stage.
- [Spline web screen](https://mobbin.com/screens/1b358622-d048-468a-a8f7-2b6f047d625e): 3D editor pattern with a large work area and dense peripheral controls.

## Applied Direction

- Keep the canvas visually dominant and avoid replacing the artwork with application chrome.
- Show live source, backend, render, inference, quality, and mode as compact state chips so performance and capture state are visible without opening another surface.
- Make source controls communicate the active input state directly.
- Improve range controls with filled tracks so tuning has immediate visual feedback.
- Keep export and performance actions in the top-right action cluster, matching creative-editor expectations for persistent capture and view controls.
